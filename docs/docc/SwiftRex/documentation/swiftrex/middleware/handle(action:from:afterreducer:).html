handle(action:from:afterReducer:)

Handles the incoming actions and may or not start async tasks, check the latest state at any point or dispatch additional actions. This is also a good place for analytics, tracking, logging and telemetry. You can schedule tasks to run after the reducer changed the global state if you want, and/or execute things before the reducer. This function is only called by the store after the receiveContext(getState:output:) was called, so if you saved the received context from there you can safely use it here to get the state or dispatch new actions.
